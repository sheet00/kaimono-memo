# AGENTS.md

## このリポジトリの前提

このリポジトリは、日常の買い物に特化した、スマホ片手操作前提の買い物リスト Web アプリを作るためのものです。

解決したい中心課題は明確です。

- Trello のような汎用ツールは、スマホでのドラッグ＆ドロップがつらい
- そのストレスをこのプロダクトでは完全に排除する
- 基本フローは次の 3 段階に固定する
  1. ストック切れに気づいたら、テンプレートから選んで買い物リストを作る
  2. 買い物中は、カゴに入れたものをチェックする
  3. 最後に固定された完了ボタンで、チェック済み項目をまとめて消す

エージェントは、ユーザーから明示的な変更指示がない限り、この操作モデルを壊さないことを優先してください。

## プロダクトの目的

目指すのは、自分用の買い物において最小の負担で使える買い物リスト体験です。重視する価値は次の通りです。

- 片手で操作できること
- ドラッグ＆ドロップが不要であること
- 文字入力を最小限にできること
- 判断コストが低いこと
- 誤操作で項目が消えにくいこと

このアプリは、汎用タスク管理ツールではありません。

## ユーザーストーリー

### 1. 気づいたとき

ユーザーとして、冷蔵庫や棚を見てストック切れに気づいたとき、テンプレートから該当アイテムをワンタップで選び、文字入力なしで今日の買い物リストへすぐ追加したい。

### 2. 買い物中

ユーザーとして、片手にカゴを持ったまま売り場を移動しているとき、カゴに入れたアイテムのチェックボックスを親指でタップし、ドラッグ操作なしで安全に状態を記録したい。

### 3. 買い物の締めくくり

ユーザーとして、必要なものをすべてチェックし終えたあと、画面下部に固定された完了ボタンを押して、チェック済みアイテムだけを一括で買い物リストから消したい。

## UX 判断の優先順位

UI や挙動を決めるときは、次の順で優先してください。

1. スマホで親指だけで扱いやすいこと
2. テンプレートから素早くリストを作れること
3. 誤削除を防げること
4. 歩きながらでも見やすく迷いにくいこと
5. 見た目がシンプルであること

実装判断に落とすと、次の方針になります。

- 操作はタップ優先にする
- ドラッグ、スワイプ、ホバー、長押し前提の UI は避ける
- 重要な操作はモバイル画面下部から届きやすい位置に置く
- チェックした瞬間に項目を自動削除しない
- 削除は「完了」という明示的な操作にまとめる
- 文字入力は補助手段とし、主役はテンプレート選択にする

## エレベーターピッチ

このプロダクトは、日常の買い物リスト管理に Trello などの汎用ツールを使っていて、スマホでのドラッグ操作につらさを感じている人向けの、片手・ワンタップ特化型の買い物 Web サービスです。

テンプレートから選ぶだけでリスト化でき、買い物中はチェックを入れ、最後に完了ボタンで一括消去できます。

一般的な Todo アプリやボード型ツールと違い、長押しやドラッグ＆ドロップが不要で、歩きながらでも誤操作なく片手で使えることが決定的な違いです。

## キャッチコピー

さよなら、売場でのドラッグ操作。片手タップで完結する、一番ミニマルな買い物リスト。

## やらないこと

シンプルさと片手操作性を守るため、ユーザーから明示的な要望がない限り、次の方向には広げません。

- 複数人でのリアルタイム同期や共有
- 「進行中」などの複雑なタスク状態管理
- 期限、通知、リマインダー
- レシート読み取り
- 価格比較
- 家計簿や支出管理との連携

## 実装ガイド

- モバイルファーストで考える
- まずは単純な状態管理と単純な画面遷移を優先する
- 情報構造はできるだけ小さく保つ
  - テンプレート
  - 現在の買い物リスト
  - チェック状態
- プロダクトの約束を壊す操作は入れない
  - 特にドラッグ＆ドロップは避ける
- 新機能を提案・実装する場合は、次のコアループにどう貢献するかを説明できるようにする
  - テンプレートから選ぶ
  - 買い物中にチェックする
  - 最後にまとめて完了する

## 現在のリポジトリ状況

- フロントエンドは `src/` 配下の Vite + React + TypeScript 構成
- 現状はまだテンプレート由来の初期コードが多い
- デモ用 UI を磨くより、買い物リスト体験へ寄せる変更を優先する

## 良い変更の基準

このリポジトリにおける良い変更は、たいてい次のどれかに当てはまります。

- タップ数や入力量を減らす
- 片手操作しやすくする
- 買い物中の進捗を見やすくする
- 誤削除や誤操作を減らす
- テンプレートからリスト作成までの流れを強くする

逆に、次のような変更は疑ってください。

- デスクトップ前提の複雑さを増やす
- ドラッグ操作や隠れたジェスチャーを導入する
- 汎用 Todo アプリ化する
- コア体験が固まる前に高機能化する

## 迷ったときの判断

要件が曖昧なときは、次の条件をもっとも強く満たす、より単純な案を選んでください。

- 自分用の買い物で使いやすい
- スマホ片手で扱いやすい
- 自動削除ではなく、明示的な完了操作を守れる
